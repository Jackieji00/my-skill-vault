# Instruction Librarian (说明书图书馆理员)

此技能用于在 Obsidian 库中自动化整理说明书文件，采用双层规则校验体系。

## 规则优先级 (Hierarchy of Rules)

1.  **全局禁忌与规范 (`/requirements.md`)**：
    *   **动作前必读**：在执行任何文件操作或网络请求前，必须阅读根目录下的 `requirements.md`。
    *   **核心用途**：了解项目级的限制（禁忌）、全局安全要求及同步规则。
    *   **注意**：即便文档提到 GitHub 上传，也需结合当前环境判断（如用户指出 Obsidian 本身不使用 Git）。

2.  **领域整理指南 (`/说明书/_Guide.md`)**：
    *   **分类逻辑**：定义品牌、型号、标签映射及具体的子文件夹路径。
    *   **命名规范**：定义说明书的最终文件名格式。

## 自动化工作流

### 1. 预检阶段 (Pre-check)
- **扫描禁忌**：读取 `/requirements.md`，识别是否有禁止的操作（如：禁止重命名、禁止移动到特定目录、禁止联网分析等）。
- **加载指南**：读取 `/说明书/_Guide.md`，理解如何处理即将扫描的文件。

### 2. 识别与扫描
- **入口**：扫描库中的 `_Pending/` 或待处理目录下的 `.pdf` 文件。
- **目标**：专注于识别“说明书”、“手册”、“User Manual”等文档。

### 3. AI 深度解析 (Extraction)
- **信息提取**：提取品牌（Brand）、型号（Model）、语言、核心功能摘要。
- **合规性检查**：根据提取的信息，再次核对 `_Guide.md` 中的具体规则。

### 4. 归档与笔记化 (Action)
- **文件重命名**：按照 `_Guide.md` 要求的格式重命名。
- **移动文件**：将 PDF 移入 `说明书/` 下的对应品牌/类别子目录。**处理完后必须清空待处理区。**
- **生成笔记**：在 PDF 同级目录下生成同名 `.md` 文件。
    - **嵌入预览**：`![[文件名.pdf]]`。
    - **结构化摘要**：记录产品型号、品牌、核心说明、重置/保养步骤。
    - **打标**：根据 `_Guide.md` 的映射关系自动打上 `#说明书/[品牌]` 等标签。

### 5. 全局合规执行
- **同步操作**：如果根目录 `requirements.md` 强制要求同步，且环境允许，则执行相关同步指令。

## 示例 `说明书/_Guide.md` 结构

```markdown
# 说明书维护指南

## 分类路径
- 数码产品: `说明书/数码/[品牌]/`
- 家用电器: `说明书/家电/`

## 标签规则
- 必须包含 `#说明书`。
- 如果包含 "乐高", 添加 `#hobby/lego`。

## 命名模版
`[品牌]-[型号]-Manual.pdf`
```

## 使用方法

`gemini invoke instruction-librarian --dir <Obsidian库路径>`
